LionAGI Graph Infrastructure¶

This module provides classes and logic for building and manipulating directed graphs within LionAGI. By representing nodes and edges as Elements (with unique IDs, timestamps, and metadata), the system enables:

  • Adding and removing nodes/edges dynamically,

  • Tracking relationships between nodes (predecessors, successors),

  • Enforcing optional edge conditions for traversal,

  • Visualizing graphs via NetworkX (if installed).

Key classes include:

  • Node: The fundamental vertex, storing arbitrary content, metadata, and optional numeric embeddings.

  • Edge: Directed link from a head node to a tail node, optionally associated with a EdgeCondition.

  • Graph: An element containing a collection of internal nodes and edges, plus mapping for quick in-/out-edge lookups.

These classes comply with LionAGI’s base Relational concept, so they can easily integrate with other LionAGI frameworks (e.g., Pile).

4. Putting It All Together¶

A typical usage pattern:

  1. Create a Graph.

  2. Add nodes (each possibly storing content, embeddings, etc.).

  3. Add edges referencing existing nodes, optionally labeling them or providing an EdgeCondition.

  4. Query the graph for predecessors/successors, or remove nodes/edges as needed.

  5. Visualize (if you have networkx + matplotlib) with Graph.display(...)().

If you want advanced logic on edges (like checking user permission or some dynamic condition), implement an EdgeCondition that returns True or False in its apply(...)`(). Then, an AI or some controlling code can call Edge.check_condition(...)`() to see if the path is allowed.

Example (short version):

from lionagi.protocols.graph.graph import Graph
from lionagi.protocols.graph.node import Node
from lionagi.protocols.graph.edge import Edge

# Setup
graph = Graph(name="MyGraph")

# Make nodes
nA = Node(content="A")
nB = Node(content="B")
graph.add_node(nA)
graph.add_node(nB)

# Make edge
eAB = Edge(head=nA, tail=nB, label=["A->B"])
graph.add_edge(eAB)

# Retrieve successors
print(graph.get_successors(nA))
# => Pile containing node B

# Visualize if you have networkx + matplotlib
# graph.display()

Summary¶

The LionAGI graph subsystem allows flexible, ID-based linking of nodes and edges, storing additional data (embedding, conditions, etc.) as needed. Combined with concurrency (e.g., Processor) or other LionAGI features, it forms the foundation for knowledge graphs, state machines, or agent-based world models.
