Pile & Progression in lionagi.protocols¶

Pile and Progression are two powerful data structures that LionAGI uses to store and manage elements in an ordered (or partially ordered) fashion. Both classes leverage LionAGI’s built-in Element and IDType for consistent, UUID-based identification of items.

Progression (progression.py)¶

A Progression represents a simple, strictly ordered sequence of item IDs. It inherits from Element (which provides it with a UUID-based identity and metadata) and from Ordering.

Key Points:

  • Stores only IDs (of type IDType) in a list, accessible via order.

  • Offers list-like operations (append, insert, slicing, etc.).

  • Offers set-like operations for IDs (+, -, intersection, difference, etc.).

  • Can be labeled with a name attribute for easy reference.

class lionagi.protocols.generic.progression.Progression(Generic[E])¶

Tracks a strict ordering of item IDs, typically referencing Element objects stored elsewhere (for example, in a Pile).

Attributes:

Methods:

  • Append / Insert / Remove:

    append(item: Any) → None¶

    Add a single ID (or item convertible to an ID) to the end of the progression.

    include(item: Any) → bool¶

    Add one or more IDs if not already present. Returns True if at least one new item was added, or False otherwise.

    exclude(item: Any) → bool¶

    Remove one or more IDs if present. Returns True if at least one item was removed, or False otherwise.

    insert(index: int, item: Any) → None¶

    Insert one or more IDs at the given list position.

    remove(item: Any) → None¶

    Remove the first occurrence of each specified ID (raises ItemNotFoundError if not found).

  • Slicing / Indexing:

    Progression supports Pythonic slicing. For example:

    subprog = my_progression[0:3]  # returns a new Progression with the first 3 IDs

  • Set-like operators:

    new_prog = prog1 + prog2   # union (concatenation) of IDs
diff_prog = prog1 - prog2  # remove IDs in prog2 from prog1

    These operations produce new Progression objects without mutating the originals.

Example:

from lionagi.protocols.generic.progression import Progression

prog = Progression(name="MySequence")

# Add IDs (these can be raw strings if they are valid UUIDs)
prog.append("8299a0ea-96d2-4514-afaa-409ece7e63d9")
prog.include([
    "35092634-3694-46c1-8f40-8ef85b349948",
    "4dbccfcc-9713-40dd-b36c-e9e29e1cbb8f"
])
print(prog.order)
# -> ['8299a0ea-96d2-4514-afaa-409ece7e63d9',
#     '35092634-3694-46c1-8f40-8ef85b349948',
#     '4dbccfcc-9713-40dd-b36c-e9e29e1cbb8f']

# Remove an ID
prog.remove("35092634-3694-46c1-8f40-8ef85b349948")

# Slicing yields either a single ID or a new Progression
first_id = prog[0]     # the first ID in the list
subprog = prog[:1]     # a new Progression with just the first item

print(subprog.order)
# -> ['8299a0ea-96d2-4514-afaa-409ece7e63d9']

Pile (pile.py)¶

A Pile manages actual objects (subclasses of Element), storing them in a dictionary keyed by their UUID, while also tracking the order of insertion via an internal Progression.

Key Points:

  • Each stored item must implement Observable (i.e., have an id field).

  • A Pile can optionally enforce type constraints via item_type.

  • Internally uses a standard dict mapping {IDType -> item} plus a Progression to remember the insertion order.

  • Provides built-in thread safety (via a standard Python lock) and async safety (via an asyncio lock) for concurrent usage.

class lionagi.protocols.generic.pile.Pile(Element, Collective[E], Generic[E])¶

Wraps a dictionary of {IDType: item}, plus a progression to maintain insertion order. Offers many synchronous and asynchronous methods (e.g., pop() vs. apop(), include() vs. ainclude(), etc.) for safe usage in multi-threaded or async contexts.

Attributes:

Thread / Async Locks:

The Pile uses an internal lock (lock) for synchronous operations and an async_lock for asynchronous operations, ensuring that concurrent modifications do not corrupt internal state.

Core Methods:

  • Include / Exclude:

    include(item)¶
    Insert one or more items by generating their ID (if necessary) and storing
    them in the dictionary. If an item's ID already exists, it is skipped. The
    insertion order is appended to :attr:`progression`.
    exclude(item)¶
    Remove one or more items (by ID) if found. No error if absent.

  • Pop / Get:

    pop(key, default=UNDEFINED)¶
    Remove and return item(s) referenced by ``key`` (which can be an ID,
    a slice, or an integer index). If not found and no default is
    provided, raises :exc:`ItemNotFoundError`.
    get(key, default=UNDEFINED)¶
    Retrieve item(s) referenced by ``key`` without removing them.
    If missing, return ``default`` or raise an error.

  • Set / List operations:

    # Union
pile_c = pile_a | pile_b
# Intersection
pile_d = pile_a & pile_b
# Symmetric difference
pile_e = pile_a ^ pile_b

    When both operands are Pile objects, these operations combine or filter the items accordingly.

  • Dumping/Loading:

    to_csv_file(fp, **kwargs)¶
    Write items (serialized by :meth:`Element.to_dict`) to CSV.
    to_json_file(fp, **kwargs)¶
    Write items to JSON, storing a dict with a ``"collections"`` key.

    A Pile can be reloaded from JSON or CSV by calling the pile() factory function with the fp parameter.

Async Methods:

Each major method (e.g., pop(), include(), etc.) has an async equivalent (e.g., apop(), ainclude()), which acquires the async_lock.

Example:

from lionagi.protocols.generic.pile import Pile
from lionagi.protocols.generic.element import Element

class MyItem(Element):
    # Extra fields, if needed
    pass

# Create items
item1 = MyItem(metadata={"tag": "alpha"})
item2 = MyItem(metadata={"tag": "beta"})

# Create a Pile enforcing type = MyItem
p = Pile(item_type={MyItem}, strict_type=True)

# Insert items
p.include(item1)
p.include(item2)

# Access by index or ID
first = p[0]         # item1
second = p[item2.id] # item2

# Slicing
slice_ = p[:2]

# Save to JSON
p.to_json_file("my_pile.json")

# Load from file
from lionagi.protocols.generic.pile import pile
p2 = pile(fp="my_pile.json")
print(len(p2))  # Should match len(p)

# Combining Piles
# union
p3 = p | p2
print(len(p3))

Comparison Overview¶

Summary¶

  • Progression is a minimal, strictly ordered list of IDs, suitable for lightweight tracking of sequences (like workflows or chain-of-thought).

  • Pile is a more robust, thread-/async-safe container, mapping IDs to actual objects. It retains insertion order through an internal Progression, supports type enforcement, and offers CSV/JSON serialization.

Together, Progression and Pile form a cohesive system for referencing and managing LionAGI objects in an orderly and concurrency-safe manner, whether you only need ID-based sequences (Progression) or a complex keyed structure (Pile).
